# 🎉 Поздравлятор

Веб-приложение для ведения списка дней рождения друзей, знакомых и сотрудников. Проект выполнен в рамках летней учебной практики.

## 📋 О проекте

**Поздравлятор** — это SPA-приложение (Single Page Application), которое позволяет:
- 👥 Просматривать список всех именинников.
- 📅 Отслеживать ближайшие дни рождения (автоматическая выборка на 7 дней вперед).
- 📷 Добавлять новых людей с загрузкой их реальных фотографий на сервер.
- ✏️ Редактировать данные (имена, даты, email, комментарии и аватары).
- 🗑️ Удалять записи из базы данных с автоматической очисткой файлов изображений.

## 🛠️ Технологический стек

### Бэкенд
- **Платформа:** .NET 10 (ASP.NET Core Web API)
- **Доступ к данным:** Entity Framework Core
- **СУБД:** PostgreSQL
- **Документация:** Swagger / OpenAPI для тестирования эндпоинтов

### Фронтенд
- **Библиотека:** React (сборщик Vite)
- **Запросы к API:** Axios
- **Стилизация:** Чистый CSS

### Архитектурные паттерны
- **Clean Architecture:** Разделение проекта на слои ответственности (Core, Infrastructure, Services, API).
- **Dependency Injection (DI):** Внедрение зависимостей для слабой связанности компонентов.
- **Асинхронность:** Использование паттерна async/await на всех уровнях.

## 🚀 Инструкция по запуску

### Требования
Перед запуском убедитесь, что у вас установлены:
- .NET 10 SDK
- Node.js (v18 или новее)
- PostgreSQL

### 1. Клонирование репозитория
Для клонирования проекта выполните в терминале:

    git clone https://github.com/Raz0Rz/Congratulations.git
    cd Congratulations/congratulator-app

### 2. Настройка базы данных
Создайте пустую базу данных в PostgreSQL (например, с именем birthdaypersons) и укажите строку подключения в файле backend/API/appsettings.json:

    {
      "ConnectionStrings": {
        "DefaultConnection": "Host=localhost;Port=5432;Database=birthdaypersons;Username=postgres;Password=ваш_пароль_от_postgres"
      }
    }

> Важно: При первом запуске бэкенда база данных и все необходимые таблицы будут созданы автоматически благодаря методу EnsureCreated(). Ручной запуск миграций не требуется.

### 3. Запуск бэкенда (API)
Перейдите в папку бэкенда и запустите сервер:

    cd backend/API
    dotnet run

- Адрес API: http://localhost:5227
- Документация Swagger: http://localhost:5227/swagger

### 4. Запуск фронтенда (React)
Откройте второй терминал, перейдите в папку фронтенда, установите зависимости и запустите локальный сервер разработки:

    cd frontend
    npm install
    npm run dev

- Адрес приложения: http://localhost:5173
